# Java Mastery Reference Project

A high-fidelity, interactive, and comprehensive codebase serving as a living reference and interview-prep resource covering the breadth of core Java—from absolute basics to advanced JVM internals and design patterns.

## Interactive Exploration

This codebase features an interactive CLI explorer to search, view details, and run examples directly from your terminal.

To launch the explorer:
```bash
mvn compile exec:java "-Dexec.mainClass=com.mastery.ConsoleNavigator"
```

To run a specific concept directly:
```bash
mvn compile exec:java "-Dexec.mainClass=com.mastery.basics.StringDeepDive"
```

To run all unit tests:
```bash
mvn test
```

To regenerate this index from source code annotations:
```bash
mvn compile exec:java "-Dexec.mainClass=com.mastery.utils.DocGenerator"
```

## Suggested Learning Paths

### Path 1: The Sequential Learner (Basics → Expert)
Best for developers studying sequentially to build a rock-solid foundation:
1. **Basics** (`com.mastery.basics.*`): Master memory caching, precision arithmetic, loops, and the String pool.
2. **OOP Foundations** (`com.mastery.oop.*`): Practice encapsulation, dynamic dispatch, abstraction interfaces, and composition.
3. **Exceptions** (`com.mastery.exceptions.*`): Learn checked/unchecked boundaries, suppressed close exceptions, and stack trace chaining.
4. **Collections & Generics** (`com.mastery.collections.*`, `com.mastery.generics.*`): Deep dive into HashMap bucket structures, specialized reference maps (WeakHashMap), and PECS generic rules.
5. **Functional & Streams** (`com.mastery.functional.*`, `com.mastery.streams.*`): Map lambda closures, currying, lazy stream pipelines, custom collectors, and parallel ForkJoinPool isolation.
6. **Concurrency** (`com.mastery.concurrency.*`): Investigate thread memory visibility (volatile), explicit locks, thread pools, async futures, and lightweight Virtual Threads.
7. **JVM Internals** (`com.mastery.jvm_internals.*`): Explore stack vs heap limits, classloading subsystems, and GC cleaning allocations.
8. **Design Patterns & JDBC** (`com.mastery.designpatterns.*`, `com.mastery.jdbc.*`): Implement interview-proof Singletons, structural design templates, SQL batch transactions, and HikariCP pooling.
9. **Interview Prep** (`com.mastery.interview.*`): Code LinkedList reversals, custom LRU caches, and monitor wait/notify queues.

### Path 2: The Quick Interview Refresh (High-Yield Topics)
If you have an interview tomorrow, jump directly to these high-yield conceptual classes:
- [ ] [SingletonPattern.java](src/main/java/com/mastery/designpatterns/SingletonPattern.java) — Double-checked locking volatile requirements; breaking and defending singletons.
- [ ] [MapsInternals.java](src/main/java/com/mastery/collections/MapsInternals.java) — HashMap bucket collision arrays and treeification threshold rules.
- [ ] [MemoryVisibilityVolatile.java](src/main/java/com/mastery/concurrency/MemoryVisibilityVolatile.java) — CPU register caching vs volatile happens-before memory visibility.
- [ ] [VirtualThreads.java](src/main/java/com/mastery/concurrency/VirtualThreads.java) — Platform threads vs Loom heap-allocated virtual threads scaling.
- [ ] [LruCache.java](src/main/java/com/mastery/interview/LruCache.java) — O(1) read/write cache using HashMap + custom Doubly-Linked Node pointers.
- [ ] [ProducerConsumer.java](src/main/java/com/mastery/interview/ProducerConsumer.java) — Thread wait/notify coordination inside while loops.
